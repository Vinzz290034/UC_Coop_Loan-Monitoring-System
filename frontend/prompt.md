## Development Guidelines

When implementing any feature or making changes, follow these guidelines:

### General
- Be precise and intentional. Generate clean, readable, and production-ready code.
- Follow best practices for maintainability, scalability, performance, and accessibility.
- Keep implementations simple unless additional complexity is clearly justified.

### Consistency
- Follow the existing architecture, coding standards, naming conventions, and project patterns.
- Maintain consistency with the current UI theme, styling, and user experience.
- Do not introduce new patterns or technologies unless necessary.

### Reuse Existing Code
- Reuse existing components, hooks, utilities, services, helper functions, constants, and types whenever possible.
- Do not duplicate functionality that already exists.
- Extend or refactor existing code instead of recreating it when appropriate.

### Project Structure
- Respect the existing folder and file structure.
- Place new files in the most appropriate existing directories.
- Keep imports, exports, and module organization consistent with the current codebase.

### Responsive Design
- Ensure all UI changes work seamlessly across:
  - Desktop
  - Tablet
  - Mobile
- Follow the existing responsive design approach.
- Avoid unnecessary responsive complexity while maintaining a polished experience on all screen sizes.

### Code Documentation
- Add concise and meaningful comments only when they explain:
  - Complex business logic
  - Non-obvious implementation details
  - Important architectural decisions
- Do not comment code that is already self-explanatory.

### Clarification
- If any requirement is unclear, ambiguous, or incomplete, ask clarifying questions before implementing.
- Do not make assumptions that could affect functionality or user experience.

### Implementation
- Consider all affected files when making changes.
- Update related types, interfaces, tests, documentation, and configuration if required.
- Ensure new code integrates seamlessly with the existing application.

### Quality Checklist
Before completing the task, verify that:
- The implementation follows the existing architecture.
- The UI matches the current design system and theme.
- Existing functionality is not broken.
- There are no unused imports, variables, functions, or files.
- The code is clean, readable, and maintainable.
- The solution is fully responsive.
- The implementation is complete and production-ready.
